## プロジェクトデータ構造

### ディレクトリ構成

```
data/projects/{project_name}/
├── project_info.json              # プロジェクト設定
├── output.xlsx                    # Excel出力ファイル
├── event_1/                       # イベント1のデータ
│   ├── master_stage.csv          # ステージマスタ（ID確定後）
│   ├── master_idolname.csv       # グループマスタ（ID確定後）
│   ├── turn_id_data.csv          # 出番データ（ID確定後）
│   ├── ライブ/                   # ライブタイムテーブル
│   │   ├── raw.png              # 元画像
│   │   ├── raw_cropped.png      # 切り出し後の画像
│   │   ├── stage_0.png          # ステージ0の画像
│   │   ├── stage_0_addtime.png  # 時刻追記済み画像（ライムライト式）
│   │   ├── stage_0.json         # 読み取り結果JSON
│   │   ├── stage_0_timetable.png # 可視化画像
│   │   └── ...
│   ├── 特典会/                   # 特典会タイムテーブル
│   │   └── ...
│   └── ライブ特典会/             # 特典会併記タイムテーブル
│       └── ...
├── event_2/
│   └── ...
└── ...
```

### project_info.json

プロジェクト全体の設定を管理するJSONファイルです。

```json
{
    "project_name": "SOUNDGARDEN2024",
    "event_type": "フェス",
    "event_num": 2,
    "ticket_urls": {
        "scope": "project",
        "urls": [
            "https://ticketdive.com/event/example",
            "https://livepocket.jp/e/example"
        ]
    },
    "event_detail": [
        {
            "event_no": 0,
            "event_name": "event_1",
            "ticket_urls": [],
            "timetables": {
                "ライブ": {
                    "format": "ライムライト式",
                    "stage_num": 4,
                    "stage_list": [
                        {"stage_no": 0, "stage_name": "ステージA"},
                        {"stage_no": 1, "stage_name": "ステージB"},
                        {"stage_no": 2, "stage_name": "ステージC"},
                        {"stage_no": 3, "stage_name": "ステージD"}
                    ],
                    "time_pixel": {
                        "time_start": "10:00",
                        "start_pix": 50,
                        "total_pix": 800,
                        "total_duration": 600
                    }
                },
                "特典会": {
                    "format": "通常",
                    "stage_num": 4,
                    "stage_list": [...]
                }
            }
        },
        {
            "event_no": 1,
            "event_name": "event_2",
            ...
        }
    ]
}
```

### フィールド説明

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `project_name` | string | プロジェクト名 |
| `event_type` | string | イベント形式（"フェス" / "対バン"） |
| `event_num` | int | イベント数 |
| `ticket_urls.scope` | string | チケットURLの紐づけ単位（"project" / "event"） |
| `ticket_urls.urls` | array | プロジェクト共通のチケットURL |
| `event_detail[].event_no` | int | イベント番号（0始まり） |
| `event_detail[].event_name` | string | イベント名（フォルダ名） |
| `event_detail[].ticket_urls` | array | イベント固有のチケットURL |
| `event_detail[].timetables` | object | 種別ごとのタイムテーブル設定 |

### timetablesオブジェクト

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `format` | string | 形式（"通常" / "ライムライト式" / "特典会併記"） |
| `stage_num` | int | ステージ数 |
| `stage_list` | array | ステージ情報の配列 |
| `time_pixel` | object | 時間軸とピクセルの対応（オプション） |

### time_pixelオブジェクト

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `time_start` | string | 基準開始時刻（HH:MM形式） |
| `start_pix` | int | 開始時刻のピクセル位置（上端からの距離） |
| `total_pix` | int | 時間範囲のピクセル幅 |
| `total_duration` | int | 時間範囲（分） |

## タイムテーブルJSONフォーマット

### 通常形式 / ライムライト式（ライブのみ）

```json
{
    "ステージ名": "STAGE-A",
    "タイムテーブル": [
        {
            "グループ名": "アイドルグループA",
            "グループ名_採用": "アイドルグループA（正式名称）",
            "ライブステージ": {
                "from": "10:00",
                "to": "10:20"
            },
            "出番ID": 1
        },
        {
            "グループ名": "架空のアイドル",
            "グループ名_採用": "架空のアイドル",
            "ライブステージ": {
                "from": "10:25",
                "to": "10:45"
            },
            "出番ID": 2
        }
    ]
}
```

### 特典会併記形式

```json
{
    "ステージ名": "STAGE-A",
    "タイムテーブル": [
        {
            "グループ名": "アイドルグループA",
            "グループ名_採用": "アイドルグループA（正式名称）",
            "ライブステージ": {
                "from": "10:00",
                "to": "10:20"
            },
            "特典会": [
                {
                    "from": "10:30",
                    "to": "11:30",
                    "ブース": "A"
                }
            ],
            "出番ID": 1
        },
        {
            "グループ名": "特典会のないアイドル",
            "グループ名_採用": "特典会のないアイドル",
            "ライブステージ": {
                "from": "10:25",
                "to": "10:45"
            },
            "特典会": [],
            "出番ID": 2
        }
    ]
}
```

### フィールド説明

| フィールド | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| `ステージ名` | string | ○ | ステージ/ブースの名前 |
| `タイムテーブル` | array | ○ | 出演情報の配列 |
| `グループ名` | string | ○ | OCR読み取り結果のグループ名 |
| `グループ名_採用` | string | × | マスタ補正後のグループ名 |
| `ライブステージ` | object | ○ | ライブの時間情報 |
| `ライブステージ.from` | string | ○ | 開始時刻（HH:MM） |
| `ライブステージ.to` | string | ○ | 終了時刻（HH:MM） |
| `特典会` | array | × | 特典会の時間情報（併記形式のみ） |
| `特典会[].from` | string | ○ | 特典会開始時刻 |
| `特典会[].to` | string | ○ | 特典会終了時刻 |
| `特典会[].ブース` | string | ○ | ブース名（なし の場合あり） |
| `出番ID` | int | × | ID確定後に付与 |

## 自動ステージ分解パラメータ

### 縦線検出パラメータ

Cannyエッジ検出とHough変換による縦線検出のパラメータです。

| パラメータ | デフォルト値 | 範囲 | 説明 |
|-----------|-------------|------|------|
| `x_edge_threshold_1` | 130 | 1-500 | Cannyエッジ検出の低閾値。検出された線が途切れる場合は下げる |
| `x_edge_threshold_2` | 285 | 1-500 | Cannyエッジ検出の高閾値。特定の色間の線が検出できない場合は下げる |
| `x_hough_threshold` | 100 | 1-500 | ハフ変換の投票閾値。全体的に検出できない場合は下げる |
| `x_hough_gap` | 1 | 0-100 | 線分の許容ギャップ（px）。連続文字上の不要な線が検出される場合は上げる |
| `x_minlength_rate` | 0.01 | 0.0-1.0 | 最小線分長（画像高さ比）。短い不要な線が検出される場合は上げる |
| `x_identify_interval` | 5 | 1-30 | 同一線とみなす横位置誤差（px） |

### 調整のヒント

1. **線が検出されない場合**
   - `x_edge_threshold_2` を下げる（色のコントラストが低い場合）
   - `x_hough_threshold` を下げる（全体的に検出が弱い場合）

2. **余分な線が検出される場合**
   - `x_hough_gap` を上げる（文字上の線を除去）
   - `x_minlength_rate` を上げる（短い線を除去）

3. **線が途切れる場合**
   - `x_edge_threshold_1` を下げる

## 横線読み取りパラメータ

### ライムライト式の横線検出パラメータ

| パラメータ | デフォルト値 | 範囲 | 説明 |
|-----------|-------------|------|------|
| `y_edge_threshold_1` | 80 | 1-500 | Cannyエッジ検出の低閾値 |
| `y_edge_threshold_2` | 150 | 1-500 | Cannyエッジ検出の高閾値 |
| `y_hough_threshold` | 60 | 1-500 | ハフ変換の投票閾値 |
| `y_hough_gap` | 1 | 0-100 | 線分の許容ギャップ（px） |
| `y_minlength_rate` | 0.05 | 0.0-1.0 | 最小線分長（画像幅比） |
| `y_identify_interval` | 5 | 1-30 | 同一線とみなす縦位置誤差（px） |
| `y_ignoretime_threshold` | 5 | 0-60 | 無視する時間幅（分以下）。この値以下の時間間隔の線は時刻を書き込まない |

### 時刻変換の仕組み

```
時刻 = 基準時刻 + (ピクセル位置 - 基準位置) × (総時間 / 総ピクセル幅)
```

**例:**
- 基準時刻: 10:00
- 基準位置: 50px
- 総ピクセル幅: 800px
- 総時間: 600分

あるピクセル位置 450px の時刻:
```
時刻 = 10:00 + (450 - 50) × (600 / 800) = 10:00 + 300分 = 15:00
```

## 出力データ構造

### Excel出力形式

`output.xlsx` の構成:

| シート名 | 内容 |
|---------|------|
| event_1 | イベント1のデータ |
| event_2 | イベント2のデータ |
| ... | ... |

### 各シートのレイアウト

```
     A          B          C          D          E          F          G          H
1  ステージID  ステージ名  特典会フラグ
2    0        STAGE-A     False
3    1        STAGE-B     False
4
5  グループID  グループ名_採用
6    0        アイドルA
7    1        アイドルB
8
9  出番ID   ライブ_from  ライブ_長さ(分)  グループID  ステージID  グループ名_raw  グループ名  ステージ名  備考
10   0       10:00        20            0          0          アイドルA      アイドルA   STAGE-A
11   1       10:25        20            1          0          アイドルB      アイドルB   STAGE-A
```

### 出番データのカラム詳細

| カラム | 型 | 説明 |
|--------|-----|------|
| 出番ID | int | 一意のID（イベント内で一意） |
| ライブ_from | string | 開始時刻（HH:MM） |
| ライブ_長さ(分) | int | 演目時間（分） |
| グループID | int | グループマスタへの外部キー |
| ステージID | int | ステージマスタへの外部キー |
| グループ名_raw | string | OCR読み取り結果 |
| グループ名 | string | 補正後のグループ名 |
| ステージ名 | string | ステージ/ブース名 |
| 備考 | string | 備考欄 |
